# 리포지토리 구조 파악

## 1. 프로젝트 개요

이 프로젝트는 **디지털 서명(Digital Signature)** 을 C 언어와 OpenSSL 라이브러리를 이용해 구현한 것입니다.  
알고리즘은 **SHA-256** (해시) 및 **RSA-2048** (비대칭 암호화)를 사용합니다.

- 운영 환경: Linux
- 언어: C
- 빌드 도구: GNU Make
- 암호화 라이브러리: OpenSSL 1.1.1

## 2. 디렉터리 구조

```
digital_signature/
├── .gitignore               # 빌드 산출물 등 추적 제외 규칙
├── Makefile                 # 최상위 빌드 스크립트 (Alice, Bob, sources 서브 디렉터리 일괄 빌드)
├── README.md                # 프로젝트 소개 및 사용법 (영문)
├── test.c                   # 초기 프로토타입/테스트용 코드 (빌드 대상 외)
│
├── keys/                    # RSA-2048 키 파일 (PEM 형식)
│   ├── Alice_private.pem    # Alice의 RSA 개인 키
│   ├── Alice_public.pem     # Alice의 RSA 공개 키
│   ├── Bob_private.pem      # Bob의 RSA 개인 키 (현재 프로그램에서 미사용)
│   └── Bob_public.pem       # Bob의 RSA 공개 키 (현재 프로그램에서 미사용)
│
├── sources/                 # 서명 대상 원본 파일 소스
│   ├── Makefile             # dummy 실행 파일 빌드
│   └── dummy.c              # 서명 대상 더미 소스 코드
│
├── Alice/                   # 서명(Signing) 프로그램
│   ├── Makefile             # signing 실행 파일 빌드
│   └── signing.c            # 서명 생성 메인 소스
│
└── Bob/                     # 검증(Verifying) 프로그램
    ├── Makefile             # verifying 실행 파일 빌드
    └── verifying.c          # 서명 검증 메인 소스
```

## 3. 각 파일 설명

### 3-1. 최상위 파일

| 파일 | 설명 |
|---|---|
| `Makefile` | `Alice`, `Bob`, `sources` 세 서브 디렉터리를 순서대로 빌드·정리하는 최상위 Makefile |
| `README.md` | 프로젝트 설명, 실행 방법, 디렉터리 구조, 참고 링크 |
| `test.c` | RSA 개인키 암호화 / 공개키 복호화 흐름을 검증하기 위한 초기 단독 테스트 코드. 최상위 Makefile 빌드 대상이 아님 |
| `.gitignore` | 바이너리(`*.o`, `*.so`, `*.exe` 등) 및 `.vscode/` 제외, `Makefile`과 확장자 있는 파일은 추적 |

### 3-2. `keys/` — RSA 키 저장소

RSA-2048 비트 키 쌍이 PEM 형식으로 저장되어 있습니다.

| 파일 | 용도 |
|---|---|
| `Alice_private.pem` | 서명 생성 시 사용 (Alice가 해시를 암호화) |
| `Alice_public.pem` | 서명 검증 시 사용 (Bob이 복호화하여 해시 비교) |
| `Bob_private.pem` | 현재 코드에서 미사용 (향후 확장 가능) |
| `Bob_public.pem` | 현재 코드에서 미사용 (향후 확장 가능) |

### 3-3. `sources/` — 서명 대상 원본

| 파일 | 설명 |
|---|---|
| `dummy.c` | `"... This is dummy source code by Alice ..."` 를 출력하는 간단한 C 프로그램. 빌드 후 `dummy` 실행 파일이 서명 대상 원본 파일이 됨 |
| `Makefile` | `dummy.c`를 컴파일하여 `dummy` 실행 파일 생성 |

### 3-4. `Alice/` — 서명 프로그램

| 파일 | 설명 |
|---|---|
| `signing.c` | 서명 생성 프로그램 메인 소스 |
| `Makefile` | `signing.c`를 컴파일하여 `signing` 실행 파일 생성 (`-lcrypto` 링크) |

**`signing.c` 동작 흐름:**
1. `../sources/dummy` 파일을 바이너리 모드로 읽음
2. 파일 데이터에 **SHA-256** 해시 적용 → 32바이트 다이제스트 생성
3. `../keys/Alice_private.pem` 에서 Alice의 RSA 개인 키 로드
4. RSA 개인 키로 다이제스트를 **암호화** → 256바이트 서명(signature) 생성  
   (`RSA_private_encrypt`, `RSA_PKCS1_PADDING`)
5. `signed_file` 출력: **[서명 256바이트] + [원본 파일 데이터]** 형태로 저장

### 3-5. `Bob/` — 검증 프로그램

| 파일 | 설명 |
|---|---|
| `verifying.c` | 서명 검증 프로그램 메인 소스 |
| `Makefile` | `verifying.c`를 컴파일하여 `verifying` 실행 파일 생성 (`-lcrypto` 링크) |

**`verifying.c` 동작 흐름:**
1. `../Alice/signed_file` 파일을 바이너리 모드로 읽음
2. 앞 256바이트 → 서명(signature), 나머지 → 원본 데이터로 분리
3. 원본 데이터에 **SHA-256** 해시 적용 → 32바이트 다이제스트 생성
4. `../keys/Alice_public.pem` 에서 Alice의 RSA 공개 키 로드
5. RSA 공개 키로 서명을 **복호화** → 32바이트 해시값 복원  
   (`RSA_public_decrypt`, `RSA_PKCS1_PADDING`)
6. 새로 계산한 해시와 복원된 해시를 **바이트 단위 비교**
   - 일치 → `[Verification Success]` 출력, `dummy` 파일 복원 및 실행 권한 부여(`chmod 0744`)
   - 불일치 → `*** File is corrupted ***` 출력 후 종료

## 4. 전체 동작 흐름

```
[Alice 측]
sources/dummy.c ──(컴파일)──▶ sources/dummy (실행 파일)
                                      │
                              SHA-256 해시 계산
                                      │
                              Alice 개인 키로 RSA 암호화
                                      │
                              signed_file 생성
                              ┌───────────────────┐
                              │ 서명(256 B)       │
                              │ 원본 데이터       │
                              └───────────────────┘

[Bob 측]
Alice/signed_file ──(읽기)──▶ 서명 + 원본 데이터 분리
                                      │
                              원본 데이터 SHA-256 해시 계산
                                      │
                              Alice 공개 키로 서명 RSA 복호화
                                      │
                              해시값 비교
                             ┌────────┴────────┐
                          일치                불일치
                             │                   │
                      검증 성공             파일 변조 감지
                      dummy 복원
```

## 5. 빌드 및 실행 방법

### 빌드

```bash
# 최상위 디렉터리에서 전체 빌드
make

# 개별 빌드
make -C sources
make -C Alice
make -C Bob
```

### 실행 순서

```bash
# 1. Alice: 서명 생성
cd Alice
./signing
# → Alice/signed_file 생성됨

# 2. Bob: 서명 검증 및 원본 파일 복원
cd ../Bob
./verifying
# → 검증 성공 시 Bob/dummy 파일 복원됨
```

### 정리

```bash
# 최상위 디렉터리에서 빌드 산출물 전체 삭제
make clean
```

## 6. 핵심 기술 및 OpenSSL API 요약

| API | 용도 |
|---|---|
| `SHA256()` | 파일 데이터의 SHA-256 해시 계산 |
| `PEM_read_RSAPrivateKey()` | PEM 파일에서 RSA 개인 키 로드 |
| `PEM_read_RSA_PUBKEY()` | PEM 파일에서 RSA 공개 키 로드 |
| `RSA_check_key()` | RSA 개인 키 유효성 검사 |
| `RSA_private_encrypt()` | 개인 키로 데이터 암호화 (서명 생성) |
| `RSA_public_decrypt()` | 공개 키로 데이터 복호화 (서명 검증) |

> **참고:** `RSA_private_encrypt` / `RSA_public_decrypt` 조합은 실제 디지털 서명의 핵심 원리로,  
> 공개 키로는 개인 키가 생성한 서명만 복호화할 수 있어 **발신자 인증** 및 **무결성 검증**이 가능합니다.
